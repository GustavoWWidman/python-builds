# Known-good upstream SHA-256 checksums, in `sha256sum`-compatible format:
#   <sha256>  <exact filename>
#
# The build scripts look up the expected checksum for the file they downloaded
# by matching the filename in the first column of the line's SECOND field.
# You can also override per-run via the OPENSSL_SHA256 / PYTHON_SHA256 env vars.
#
# IMPORTANT (honesty note): verify these values yourself before the first real
# CI run. The OpenSSL value is the widely-published 1.1.1w release hash; the
# Python value should be cross-checked against the md5/sigstore data on
# https://www.python.org/downloads/release/python-3615/ . If you bump either
# version input, you MUST add the new filename+hash line here (or pass the
# matching *_SHA256 env var) or the build will fail closed.
